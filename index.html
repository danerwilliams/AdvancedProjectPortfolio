AR Project Phase 1

Phase 1 of the AR project was research oriented. The goal of the AR project was to create a form for the Academic Resources department at Eastside Catholic, which would automate the process students, teachers, and AR staff go through to set up testing with accommodations at school. The original process involved the student going to the AR classroom to grab a form, then the student would fill out the form with their name, the class, period, teacher, and requested accommodations for the specified test. After filling out the form, the student would then need to get the teacher to sign the form, give the form back to the student, so that the student could give the form to AR, who would then organize the form with the many others by hand. My project aimed to create a website which would make this process much more efficient and convenient for the AR staff, teachers, and students.

Resources:
Microsoft Outlook Calendar REST API
Java
Servlet / Tomcat
JSP
Maven

Conclusion:
By the end of the research phase of the AR Project had developed skills which I did not previously have in REST API's, HTTP protocol, servlet, tomcat, jsp, and maven. Most importantly I had developed a clear idea of how I would approach creating the application after developing a greater understanding of how web applications work, particularly those written with java and jsp.

AR Project Phase 3

Phase 3 of the AR Project was the third and final phase of the project. This involved deploying the application so that it can be used by Academic Resources and their students. Over the course of this phase, many deployement options were experiemented with, including many options available on a local server sucha as Apache and IIS, but eventually it was deployed in the cloud with Amazon Web Services.

Resources:

Amazon Web Services EC2 Elastic Beanstalk

Conclusion:
In phase 3 of the AR project I became very familiar with the multitude of deployment options available for web applications, as I had around 3 months of trial and error attempting to deploy my application. Eventually, I settled on a cloud service, specifically from Amazon, for its superiority to a personal server in convenience and scalability, especially compared to a local server run on a restricted school network. Additionally, I made small design changes to the application itself as I learned how certain elements can be more optimal for deployement in a cloud based web service.
